# 📋 작업 브리프

**원 명령:** 웹페이지 ui부터 결정합시다.

## 요약
웹페이지의 사용자 인터페이스(UI) 및 핵심 와이어프레임을 설계합니다. 웹페이지는 'Gap 인식'을 유도하여 유료 컨설팅 리포트로 전환시키는 고효율 MVP 랜딩 페이지가 되어야 합니다.

## 분배
- **📱 Secretary**: 웹페이지의 목표를 명확히 정의하고, 사용자가 따라야 할 핵심 플로우(User Flow)를 단계별로 요약합니다. (1. 후크/Pain Point 제시 $\rightarrow$ 2. 진단 CTA 유도 $\rightarrow$ 3. Gap 인식 강화 $\rightarrow$ 4. 프리미엄 해결책 Pitch)
- **✍️ Writer**: 랜딩 페이지의 각 섹션별 카피라이팅(Copywriting) 초안을 작성합니다. 특히, 사용자가 자신의 'Gap'을 스스로 느끼게 만드는 강력한 후크 헤드라인과 CTA 버튼 문구 3가지 세트를 제공해야 합니다.
- **🎨 Designer**: Secretary와 Writer가 정의한 플로우를 기반으로 저화질(Low-Fidelity) 와이어프레임을 구축합니다. '전문성'과 '신뢰도'를 최우선 컬러 팔레트 및 타이포그래피로 구현하고, 모든 섹션의 배치 순서와 시각적 계층 구조를 결정해주세요.
- **💻 Developer**: MVP 웹페이지에 반드시 포함되어야 할 사용자 데이터 입력 필드(Form Fields) 목록을 정의합니다. (예: 학년, 모의고사 점수, 목표 대학 등). 이 데이터들이 백엔드에서 어떤 규칙셋(Ruleset)으로 처리되고 연동될지 구조적으로 설계해주세요.
